### Agent 智能平台（Agent-factory）

**项目描述**：企业级 Agent 开发与运行平台，支撑多场景智能体的构建、编排、评测与持续优化。负责 Agent 核心工作流编排、流式对话链路设计、长期记忆（Memory）机制工程化落地及自动化评测体系建设，具备大模型应用从平台架构到高性能交互落地的完整前端工程化能力。

**技术栈**：React 18 + Next.js 14 + TypeScript + Zustand + React Query + SSE/WebSocket + Flowise + Shadcn/ui

**工作职责**

**一、Agent 工作流编排与流式交互**

- 负责 Agent 核心**工作流引擎的可视化实现**，基于 Flowise 构建可配置化工作流编排能力，将业务逻辑抽象为可配置节点
- 深度优化基于 **SSE/WebSocket** 的流式对话链路，实现 **Thought 链**、多模态内容与**代码沙箱**的实时流式渲染
- 解决复杂推理任务的可视化边界问题，支撑业务从单一 Chat 向**复杂多步 Agent 协作**的工程转型

**二、Agent 长期记忆机制与状态治理**

- 横向调研 **MemOS、LangGraph/LangMem、Letta、Mem0、Zep** 等 6 大主流 Agent 记忆框架，输出技术选型报告，主导前端工程化落地方案
- 实现**核心常驻记忆层**（用户画像 / 偏好 / Agent Profile）与**检索型记忆库**的双层架构，将高频稳定信息从纯检索对象升级为常驻上下文块
- 落地**混合检索策略**（向量 + 全文联合召回 + 时间衰减排序），替代纯向量单一检索方案，提升长上下文场景下的召回精度
- 实现**后台沉淀机制**（热路径轻写入 + 后台 consolidation），将碎片化 episodic 记忆归纳为结构化 semantic 记忆
- 引入**轻量实体层**（entities 字段 + 实体过滤与聚合），为记忆增加关系维度，支撑跨轮次的实体关联查询
- 建立**基于状态机的对话上下文管理规范**，解决多智能体协同下的状态同步与信息隔离难题

**三、Agent 评测工程与 LLMOps 治理**

- 基于 Flowise 工作流构建**自动化评测体系**，覆盖**能源助手、智慧巡检**等核心业务场景
- 设计评测数据面板与**可视化闭环**，支撑大模型输出效果的持续监测与迭代优化

**四、AI 技能生态与工具调用（SkillsHub）**

- 负责 Agent 插件调用（**Function Calling**）体系的前端集成，对接 **MCP (Model Context Protocol)** 等底层服务
- 提供高扩展性的**技能调度与可视化前端编排**能力，连接大模型与各类业务工具链
